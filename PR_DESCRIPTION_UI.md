# 🎨 フロントエンドUI実装 - モダンな3x3グリッドデザイン

## 📋 概要

過去のNuxt.js実装を参考にしつつ、モダンで使いやすいReact UIを実装しました。3x3グリッドでアルバムを選択・管理できるインターフェースを提供します。

## ✨ 主な機能

### 1. 3x3アルバムグリッド
- 9つのスロットでアルバムを管理
- 空のスロットにはプラスボタンを表示
- 選択済みアルバムには番号と情報を表示

### 2. アルバム検索・選択機能
- 各スロットのプラスボタンから検索ダイアログを開く
- Spotify API連携によるリアルタイム検索
- デバウンス処理（500ms）でAPI呼び出しを最適化

### 3. アルバム管理機能
- **追加**: 空のスロットをクリックして検索・選択
- **入れ替え**: 既存アルバムをクリックして「入れ替え」ボタンで変更
- **削除**: ホバー時に表示される削除ボタンで削除

### 4. レスポンシブデザイン
- スマホサイズでも使いやすいレイアウト
- タッチ操作に対応
- フォントサイズとボタンサイズを画面サイズに応じて調整

### 5. ローカルストレージ連携
- 選択したアルバムを自動保存
- ページリロード後も状態を保持
- ハッシュタグ設定も保存

## 🎯 実装詳細

### コンポーネント構成

#### `App.tsx`
- メインアプリケーションコンポーネント
- アルバム状態管理（9つのスロット）
- ユーザー名・タイトル・ハッシュタグ入力
- 投稿作成機能

#### `AlbumGrid.tsx`
- 3x3グリッド表示
- 空スロットと選択済みスロットの表示
- ホバー時の操作ボタン表示

#### `AlbumSearch.tsx`
- Spotify検索ダイアログ
- リアルタイム検索結果表示
- ローディング・エラー状態の処理

#### タイポグラフィ

#### アニメーション
- ホバー時のスムーズなトランジション
- フェードイン・フェードアウト効果

## 📱 スマホ対応

### レスポンシブブレークポイント
- **モバイル**: 3列グリッド、コンパクトなボタン
- **タブレット以上**: より大きなボタンとスペーシング

### タッチ操作
- タップで検索ダイアログを開く
- 長押しで削除（将来実装予定）
- スワイプでスクロール

## 🔗 バックエンド連携

### APIエンドポイント
- `GET /api/search?q=検索クエリ` - Spotify検索
- `POST /api/posts` - 投稿作成

### データフロー
1. ユーザーが検索キーワードを入力
2. フロントエンドがバックエンドAPIを呼び出し
3. バックエンドがSpotify APIに問い合わせ
4. 検索結果をフロントエンドに返却
5. ユーザーがアルバムを選択
6. 投稿作成時に全アルバム情報を送信

